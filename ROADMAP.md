## 1. プロジェクト概要

- **ikura** = 割り勘・立替精算 Web アプリ。Walica を参考に自作したものの一般公開版。
- 元リポジトリ: `~/Desktop/warikan_2`(身内用・現役稼働中)。本リポジトリ(`~/Desktop/ikura/new-app`)はそのフォークで、こちらを公開版として開発する。
- 差別化コンセプト: **広告なし・画面遷移が少ない・データを売らない**。「広告を載せない」はコストではなく商品の一部。
- 目標: まず「最初の1円」を稼ぐ。月数千円〜1万円規模の小銭稼ぎが現実的な期待値(課金率相場 1〜3%)。

## 2. 収益化の決定事項(2026-07-05 確定)

採用するのは以下の2つ。**広告・アフィリエイトは不採用**(体験を損なうため。オーナーの強い意向)。

### D. チップ(支援)機能 — 最優先。最初の1円はここから
- **表示場所**: 精算結果画面の「一人一人の請求額一覧」の**最下段**に1行追加する形。
- **表示仕様**: 「＋1人◯円で ikura を支援」のように**1人あたり金額**で見せる。
  - ◯ = `ceil(200 ÷ グループ人数)` 円。グループ合計が **200円** になる設計。
  - 例: 4人グループ → 「1人50円」、5人 → 「1人40円」、2人 → 「1人100円」。
  - 狙い: 1人が出す金額を小さく見せる心理設計。旅行・飲み会中は金銭感覚が緩むタイミングでもある。
- **決済の実体**: Stripe Payment Link で **1決済200円をまとめて1人(幹事)が払う**。表示上「1人◯円」だが、メンバー個別に少額決済させない(決済回数を増やさない)。
  - Stripe 手数料 3.6% → 200円あたり手取り約192円。最低決済額(50円)もクリア。
  - Apple Pay / Google Pay を有効化し「タップ2回で払える」状態にすること。カード番号手入力を要求したら負け。
- サーバー実装ゼロで置ける(Payment Link の URL を貼るだけ)。webhook 不要。

### A. プレミアムグループ(買い切り 400円)
- 無料グループ: 全機能使えるが **データは90日で消える**。
- プレミアム(400円/グループ・買い切り): 履歴の**永久保存**、精算レポートの PDF/CSV 出力、(将来)カスタムアイコン・テーマ。
- 訴求は機能ではなく**思い出の保存**。「この旅行の記録を残す?」を精算完了画面で提案(事後課金。感謝がピークの瞬間に請求する)。
- 技術: Stripe Payment Link に `client_reference_id` でグループIDを渡し、**Supabase Edge Function 1本**で Stripe webhook(`checkout.session.completed`)を受けて `groups.is_premium = true` を立てる。必要なサーバーコードはこれだけ。
- **履歴機能は削除しない**(当初削除予定だったが、これが最大の売り物なので方針転換済み)。

### 不採用・保留
- レシートOCR: 良い差別化候補だが**当面不要**(オーナー判断 2026-07-05)。将来のプレミアム機能候補として保留。
- ルーレット(次のご飯・旅行先レコメンド): アフィリエイト前提だったため保留。
- サブスク: 匿名・URL共有型と相性が悪いので当面やらない。常設グループ(シェアハウス等)市場に出るときに再検討。
- 送金機能: **永久に実装禁止**。実装すると資金移動業ライセンスが必要になる法的な崖。ikura は「計算するだけ」を維持する。

## 3. 技術スタックの決定事項

| 項目 | 決定 | 理由 |
|---|---|---|
| フロント | Vue 3 + Vite(現状維持) | 動いているものは動かさない |
| DB | Supabase(現状維持) | 無料枠で商用OK。ただし RLS 必須 |
| ホスティング | **Cloudflare Pages に移行** | Vercel Hobby は規約で商用利用禁止。Pro($20/月)は経済性が崩壊。Cloudflare Pages は無料で商用OK・帯域無制限 |
| 決済 | Stripe(Payment Link 中心) | 個人で開設可。手数料3.6% |
| 計測 | Cloudflare Web Analytics | 無料・Cookieバナー不要 |

### セキュリティ前提(重要)
- 過去コミットに `.env` が含まれるが、中身は Supabase URL と anon キーのみ = **ブラウザ配信される公開情報なので問題なし**。service_role キーは含まれていない(確認済み 2026-07-05)。
- その代わり **DB を守るのは RLS だけ**。公開前の RLS 整備は絶対条件。
- service_role キーは今後も絶対にフロント・リポジトリに置かない(Edge Function の環境変数のみ)。

  - ビルド環境変数(VITE_SUPABASE_*)は プロジェクト Settings → Build → Variables and secrets に設定済み

## 6. 課金設計の原則(判断に迷ったらここに戻る)

1. **全員からではなく幹事から取る**(決済は1グループ1回)
2. **感謝がピークの瞬間(精算完了直後)に提案する**(事前課金の壁は作らない)
3. **「払わないと不便」ではなく「払うと嬉しい」**(無料版だけで完結して満足できること)

## 7. 法務・運用メモ

- 有料機能を載せる前に特商法表記が必須(Stripe の規約上も必要)
- 収益が年20万円を超えたら雑所得として確定申告(当面心配不要)
- Supabase 無料枠: 500MB DB / 月間 MAU 5万。超える規模になったら嬉しい悲鳴として再検討
